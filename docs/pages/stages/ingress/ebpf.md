# eBPF Ingress

`EBPFStage` reads records from an eBPF ring buffer and decodes each record into
an `EBPFMessage[T]`.

``` go
cfg := ingress.NewEBPFConfig(loadBpf, attachBpf, ringBuffer)
stage := ingress.NewEBPFStage(out, cfg)
```

## Messages

### Output Message

Produced body type: `*ingress.EBPFMessage[T]`.

| Field | Description |
| --- | --- |
| `Data` | Parsed ring-buffer record as the user-provided Go type `T`. |

Additional interfaces: none. `EBPFMessage[T]` only implements the standard
`message.Body` contract.

## Configuration

| Field | Default | Description |
| --- | --- | --- |
| `LoadFn` | required | Function generated by `bpf2go` that returns an eBPF collection spec. |
| `LinkFn` | required | Function that attaches loaded objects to a hook and returns a `link.Link`. |
| `RingBufferGetter` | required | Function that returns the ring-buffer map from loaded objects. |
| `UseUnsafe` | `false` | If true, decode records with an unsafe struct cast instead of `binary.Read`. |
| `CollectionOptions` | `nil` | Optional Cilium eBPF collection load options. |

## Internals

The stage uses `github.com/cilium/ebpf`, `link`, `ringbuf`, and `rlimit`.
Initialization removes memlock limits, loads and attaches the eBPF objects,
opens a `ringbuf.Reader`, then decodes each `RawSample`. The safe path uses
`encoding/binary.Read` with little endian byte order.
